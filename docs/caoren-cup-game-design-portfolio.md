# Caoren Cup CS2 自定义娱乐赛系统 - 游戏策划作品集

## 作品集定位

**项目类型**：多人竞技玩法策划 / 活动玩法设计 / 个人实战项目

**项目一句话**：

Caoren Cup 是一个面向 CS2 社区服务器的自定义娱乐赛系统，核心目标是在传统 CS2 对抗基础上加入隐藏身份、非对称竞技和可调节数值规则，让玩家在朋友局和社区赛中获得更强的娱乐性、博弈感和复盘空间。

**适合投递方向**：

- 游戏策划
- 玩法策划
- 系统策划
- 活动策划
- 战斗 / 数值策划助理

**关键词**：

`多人竞技`、`隐藏身份`、`信息差`、`阵营博弈`、`非对称竞技`、`动态 Buff`、`数值补偿`、`玩家反馈`、`玩法迭代`、`真实服务器验证`

## 核心玩法一：卧底模式

### 设计目标

在传统 CS2 阵营对抗中加入隐藏身份和信息差，让玩家除了枪法和战术外，还需要通过行为、战绩、任务完成情况和赛后发言判断队伍中的卧底。

### 玩法设计

卧底模式包含三类身份：

- **士兵**：正常参与对抗，目标是帮助队伍赢下比赛。
- **卧底**：拥有隐藏身份和特殊任务，需要在不暴露自己的情况下完成任务。
- **侦探**：观察比赛过程，在赛后帮助队伍判断卧底身份。

卧底任务采用九宫格任务板形式，支持任务提示、任务替换、N 值任务和连线计分。赛后进入指认阶段，玩家根据局内表现和任务线索进行判断，再进入积分结算。

### 策划亮点

- **信息差设计**：卧底知道自己的隐藏目标，普通玩家只能通过行为推断身份。
- **社交推理**：玩家需要在局内观察队友行为，并在赛后进行讨论和指认。
- **行为博弈**：卧底不能只追求击杀，还要平衡任务完成、伪装和团队贡献。
- **复盘空间**：赛后可以围绕任务、指认和得分展开复盘，增强娱乐赛记忆点。

## 核心玩法二：不平衡竞技 / 一人成军模式

### 设计目标

解决社区服务器中常见的“人数不均也想开打”的问题。玩法允许双方人数不对等，并通过少人数方 Buff 补偿战斗力，让弱势方获得反打空间，同时保留人数优势方的压迫感。

### 玩法设计

不平衡竞技的核心不是简单让少人数方变强，而是根据人数差、对局阶段和玩家体感调整补偿强度。

当前玩法以“一人成军”模式为代表：

- 少人数方作为 **Solo / 独勇者**。
- 独勇者可以获得更高血量、护甲、移动速度、伤害倍率、击杀回血和阶段性雷达信息。
- 多人数方作为挑战者，拥有团队人数优势，但可通过装备限制、规则限制或数值差异控制强度。
- 模式支持换边，让不同玩家体验强势方和挑战方。

### 动态 Buff 设计思路

我的设计目标是让少人数方“有机会赢”，但不是“无脑强”。

因此 Buff 调整会参考三个维度：

1. **人数差**：人数差越大，少人数方需要更强的生命、伤害或信息补偿。
2. **局势变化**：如果少人数方经常被快速集火秒杀，需要提高生存或信息能力；如果少人数方压制过强，则降低伤害、回血或视野优势。
3. **玩家反馈**：根据玩家对“打不过”“太赖”“有反打但不离谱”等体感反馈，持续调整补偿强度。

### 策划亮点

- **动态平衡意识**：Buff 不是固定堆数值，而是根据实际局势和玩家反馈调整。
- **公平感控制**：弱势方有反打空间，人数优势方仍然觉得自己的配合和包夹有价值。
- **对局张力**：少人数方拥有高压操作空间，多人数方需要协作围剿，双方体验差异明显。
- **可迭代参数**：血量、护甲、速度、伤害、回血、雷达信息和装备限制都可以作为调参抓手。

### 简历可用表述

设计少人数阵营动态 Buff 机制，根据人数差、局势变化和玩家反馈迭代补偿强度，在保证公平感的同时提升弱势方反打空间与对局张力。

## 活动流程设计

Caoren Cup 不只是单个玩法插件，也包含完整的娱乐赛流程：

```text
大厅等待
选择队长
Roll 点决定先后手
队长选人
地图 Ban / Pick
阵营选择
赛前身份和任务配置
正式比赛
赛后指认
积分结算
```

这个流程的设计重点是让玩家知道自己当前处于哪个阶段、下一步要做什么，以及比赛结束后为什么得到这个分数。

### 策划价值

- 将零散的娱乐玩法整理成可重复组织的活动流程。
- 降低玩家理解成本，减少管理员临场解释压力。
- 让身份、任务、战绩、指认和结算形成完整闭环。

## 迭代方法

我在实际服务器游玩中通过以下方式迭代玩法：

- 观察玩家是否能快速理解规则。
- 记录哪类 Buff 容易造成碾压或无力感。
- 收集玩家对强度、公平感和娱乐性的反馈。
- 调整数值补偿、装备限制、任务说明和局内提示。
- 在后续对局中继续验证调整效果。

典型迭代问题：

- 少人数方是否死得太快，导致没有操作空间？
- Buff 是否过强，导致人数优势方觉得配合没有意义？
- 卧底任务是否太明显，导致隐藏身份过早暴露？
- 任务提示是否清楚，玩家是否知道自己该做什么？
- 赛后计分是否能解释玩家的贡献？
