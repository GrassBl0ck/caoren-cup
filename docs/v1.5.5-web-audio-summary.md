# v1.5.5 网页音效/音乐模块对话总结

本文整理 v1.5.5 开发中关于网页音效/音乐模块的讨论结论、已完成改动和后续注意事项。

## 1. 当前目标

v1.5.5 的主要内容是网页端音效/音乐模块，重点包括：

- 统一管理网页 BGM。
- 除正式比赛中以外，其他阶段都允许播放 BGM。
- 管理员可以随时向所有网页玩家发送提示音。
- 管理员提示音尽量绕过网页内开关和音量限制，保证提示效果。

## 2. BGM 播放规则

讨论后确定的规则：

- 只有 `LiveGame` 正式比赛开打中不播放 BGM。
- 其他阶段都允许播放 BGM。

允许播放 BGM 的阶段包括：

- `Lobby`：大厅 / 等待阶段
- `CaptainSelection`：队长选择
- `Roll`：Roll 点
- `PlayerDraft`：人员选择
- `MapBan`：地图 BP
- `SidePick`：最终选边
- `PreGameSetup`：赛前准备
- `MidGameQA`：中场问答
- `PostGameAccusation`：赛后指认
- `Scoreboard`：积分结算

不播放 BGM 的阶段：

- `LiveGame`：正式比赛中

## 3. BGM 管理方式

原来是按阶段分别配置 BGM，例如 `PlayerDraft`、`MapBan`、`SidePick` 各自有曲目。

现在改为统一管理：

- 所有允许播放 BGM 的阶段共用同一份 `music` 列表。
- 默认播放曲目由 `defaults.musicTrack` 决定。
- 玩家可以在网页右下角音频面板里选择自己浏览器播放哪一首 BGM。
- 玩家选择只保存在自己的浏览器 `localStorage`，不会影响其他玩家。

配置文件：

```text
web-command-center/public/assets/audio/manifest.json
```

音乐文件目录：

```text
web-command-center/public/assets/audio/music/
```

当前预留曲目：

```text
common-bgm-01.mp3
common-bgm-02.mp3
```

## 4. 多个音乐文件怎么加

如果有多个 BGM，把文件放到：

```text
web-command-center/public/assets/audio/music/
```

然后在 `manifest.json` 的 `music` 数组里增加条目，例如：

```json
{
  "id": "common_bgm_03",
  "title": "统一 BGM 03",
  "src": "/assets/audio/music/common-bgm-03.mp3",
  "loop": true
}
```

默认播放哪首由下面字段控制：

```json
"musicTrack": "common_bgm_01"
```

## 5. 管理员全员提示音

新增功能：

- 管理员控制区增加按钮：`发送提示音`。
- 点击后，前端发送 `ADMIN_ACTION: PLAY_AUDIO_CUE`。
- 后端确认当前用户是管理员。
- 后端向所有网页客户端广播 `AUDIO_CUE`。
- 所有打开网页的客户端播放 `adminPrompt`。

提示音文件目录：

```text
web-command-center/public/assets/audio/sfx/
```

需要放入实际文件：

```text
admin-prompt.mp3
```

配置项：

```json
"adminPrompt": {
  "title": "管理员提示音",
  "src": "/assets/audio/sfx/admin-prompt.mp3",
  "forceVolume": 1
}
```

## 6. 提示音限制说明

管理员提示音已尽量绕过网页系统内限制：

- 不受“启用音乐”开关影响。
- 不受 BGM 停止状态影响。
- 不受当前比赛阶段影响。
- 不受网页音量滑块影响。
- 按 `forceVolume: 1` 满音量播放。
- 玩家只要点过网页任意位置或按过键，页面会自动尝试解锁音频。

无法绕过的限制：

- 浏览器从未获得过用户交互时，可能拦截自动播放。
- 浏览器标签页被静音时，网页无法强制取消静音。
- 系统音量为 0 或输出设备问题，网页无法控制。

因此实际结论是：

网页内限制已尽量绕过；浏览器和系统级限制不能被网页代码强行突破。
