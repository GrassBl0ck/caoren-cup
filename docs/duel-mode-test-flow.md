# 单挑模式测试流程

这份流程用于单挑模式上线前验证，目标是把风险分成三层处理：

1. 本地编译检查：确认代码和插件能构建。
2. 网页大厅流程：确认临时管理员、投票、审批、旁观者规则正常。
3. 真实 CS2 实战：确认服务器命令、锁边、枪械和结算正常。

涉及服务器覆盖、重启、打开浏览器或客户端实测前，需要先征得确认。

## 0. 本地必测

推荐直接运行：

```powershell
cd D:\OpenSourcework\caoren-cup-open-source
.\scripts\test-duel-mode-local.ps1
```

如果本次也改了 `game-plugin/`，运行：

```powershell
.\scripts\test-duel-mode-local.ps1 -IncludeGamePlugin
```

脚本会执行：

```powershell
cd web-command-center
npm run typecheck
node --check public\js\lobby-app.js

cd CaorenCupPlugin
dotnet build
```

如果包含娱乐插件本体，还会执行：

```powershell
cd D:\OpenSourcework\caoren-cup-open-source\game-plugin
dotnet build
```

通过后再打测试包，不使用旧的 `release-output` 包。

## 1. 有其他人一起测试

建议至少准备 4 个网页身份：

- 管理员
- 玩家 A
- 玩家 B
- 玩家 C

### 1.1 网页大厅流程

逐项确认：

- [ ] 管理员不在大厅时，玩家 A 发起临时管理员投票，玩家 B 同意，全票通过后玩家 A 成为临时管理员。
- [ ] 投票期间玩家 C 加入，投票立刻终止。
- [ ] 投票 30 秒无人全票通过，投票失败。
- [ ] 管理员在大厅时，玩家 A 只能提交申请，管理员同意后玩家 A 成为临时管理员。
- [ ] 管理员在大厅且已有临时管理员时，管理员不能直接操作单挑设置，只能收回临时管理员。
- [ ] 管理员收回临时管理员后，玩家 A 权限消失，管理员重新承担职责。
- [ ] 有临时管理员后，新加入玩家只能是旁观者，本局不能参赛。
- [ ] 临时管理员可以设置地图、A/B 队、回合分配。
- [ ] 回合总数小于 30 时，被拒绝或回退到合法值。
- [ ] 默认回合显示为手枪 8、步枪 16、狙击 12。

### 1.2 真实 CS2 实战

至少 2 名真实玩家进服，最好 3 名：A、B、旁观者。

逐项确认：

- [ ] 开局后服务器执行单挑配置：地图正确，`mp_maxrounds` 等于 X+Y+Z，不开启半场换边。
- [ ] 网页分到 A/B 的玩家，在游戏内确实进入相对两边，不出现网页不同边但游戏同队。
- [ ] 第 1 回合默认：USP-S，无主武器。
- [ ] 手枪阶段 `/guns` 显示手枪列表，输入 `/deagle` 后下回合生效。
- [ ] 步枪阶段默认：AK47 + USP-S。
- [ ] 步枪阶段输入 `/awp` 后，对方所有参赛玩家必须 `/agree_awp`，同意后下回合才给 AWP。
- [ ] 狙击阶段默认：AWP，无副武器。
- [ ] 狙击阶段 `/guns` 只显示 `/ssg | /awp`。
- [ ] 分数未到总回合时不能提前结算。
- [ ] 打满 X+Y+Z 回合后结束。
- [ ] 同分显示平局。
- [ ] 游戏结束但网页未结束时，临时管理员点结束游戏，能进入赛后页并开下一局。
- [ ] 中途强制终止时，临时管理员输入 `TERMINATE`。
- [ ] 如果管理员在线，强制终止需要管理员同意。
- [ ] 强制终止通过后不进赛后页，效果等同强制清局。

### 1.3 开玩前最终确认

- [ ] `git status --short --ignored` 中没有准备提交的备份、`bin/`、`obj/`、zip、日志。
- [ ] 测试包内容不包含 `node_modules/`、备份、日志、`.env`、`caoren_config.json`。
- [ ] 服务器部署后网页能打开。
- [ ] 桥接插件已加载。
- [ ] CS2 服务器能收到网页命令。

## 2. 只有一个人测试

### 2.1 能完整测的部分

用多个浏览器身份模拟多人网页：

- 普通窗口登录管理员。
- 无痕窗口登录玩家 A。
- Edge、Chrome 或手机浏览器登录玩家 B。
- 再开一个窗口登录玩家 C。

可以完整确认：

- [ ] 临时管理员投票、申请、审批、收回。
- [ ] 投票 30 秒超时。
- [ ] 新玩家加入取消投票。
- [ ] 有临时管理员后新玩家变旁观者。
- [ ] 地图、回合分配、总回合 >= 30 校验。
- [ ] 管理员和临时管理员权限互斥。
- [ ] 终止游戏的网页审批流程。

### 2.2 只能做烟雾测试的部分

一个 Steam 账号不能真实模拟两个对手，所以游戏内只能确认：

- [ ] 插件能加载。
- [ ] 单挑模式配置命令能下发。
- [ ] 地图能切换。
- [ ] `/guns` 能显示。
- [ ] 当前阶段默认武器大体正确。
- [ ] `/usp`、`/ak`、`/ssg`、`/awp` 等命令不会报错。
- [ ] 结束或 `TERMINATE` 不会卡死网页流程。

不能完全确认：

- 对方所有玩家 `/agree_awp` 的多人同意逻辑。
- 两边真实玩家锁边是否 100% 正确。
- 多人加入后全程旁观是否在真实 CS2 内完全一致。

这三项必须等至少另一个人上线做一次短测。

### 2.3 单人快速安全测试顺序

1. 运行 `.\scripts\test-duel-mode-local.ps1`。
2. 网页开 3-4 个身份，跑完临时管理员流程。
3. 部署到测试服或低峰期正式服。
4. 自己进 CS2，确认插件、地图、枪械命令、结束流程。
5. 记录需要多人复测的点：AWP 同意、真实两边锁边、多人加入旁观。
6. 找到第二个人后，只补测这些点和一次打满回合测试。

## 3. 最小通过标准

正式开玩前必须满足：

- [ ] 本地编译检查全过。
- [ ] 网页端临时管理员流程全过。
- [ ] 真实服务器能切地图、锁总回合、发默认枪。
- [ ] 新加入玩家不会误进参赛队。
- [ ] 强制终止不会卡住下一局。
- [ ] 至少一次双人实测通过：两边锁边正确，AWP 需要对方同意。

如果只有一个人测，最多只能算“可部署候选”，不要直接当成“多人实战已验证”。
